# Zunery Nexus — 빠른 시작 및 사용 가이드

Zunery Nexus에 오신 것을 환영합니다. 이 안내서는 개발자가 아닌 일반 사용자를 위해 작성되었으며, 설치부터 기본 사용 방법까지 쉽게 따라하실 수 있도록 구성되어 있습니다.

설치
- macOS / Windows / Linux: GitHub Releases에서 설치 프로그램(.dmg, .exe, AppImage 등)을 다운로드하여 설치하세요.

첫 실행
- 앱을 실행하면 왼쪽 사이드바(메뉴)와 오른쪽 채팅 영역이 표시됩니다.
- 에이전트 생성: 상단의 "Agents" 메뉴로 이동해 "+ New Agent" 버튼을 눌러 새 에이전트를 만듭니다. 이름과 모델(또는 공급자)을 선택하고, 시스템 프롬프트로 에이전트의 동작(톤, 역할 등)을 지정할 수 있습니다.
- 대화 시작: "Chat" 화면으로 가서 에이전트를 선택한 뒤 "+ New Conversation"을 눌러 새 대화를 시작하거나 기존 대화를 선택해 이어서 진행할 수 있습니다.

UI 구성
- 사이드바: Chat, Agents, Settings로 전환하고 선택한 에이전트의 대화 목록을 확인할 수 있습니다.
- 채팅 화면: 메시지 기록이 중앙에 표시되며, 하단 입력창에 작성 후 Enter를 눌러 전송합니다. Shift+Enter는 줄 바꿈입니다.
- 에이전트 편집: 에이전트의 이름, 모델, 시스템 프롬프트, 온도(temperature) 및 사용 가능한 도구(웹 검색 등)를 설정합니다.
- ADK 토글: ADK 통합이 가능하면 상단 바에 "ADK" 토글이 표시됩니다. 켜면 ADK 서버를 통해 에이전트를 실행합니다.

에이전트 사용 방법
- 용도에 따라 여러 에이전트를 만드세요(예: 요약 전문가, 코드 어시스턴트).
- 시스템 프롬프트에 역할과 출력 형식을 명확히 적으면 더 일관된 응답을 얻을 수 있습니다.
- 도구 사용: 에이전트가 웹 검색 같은 도구를 사용할 때, 결과가 메시지 아래의 확장 카드로 표시됩니다.

대화 저장 및 내보내기
- 대화는 로컬에 저장되며, 현재 버전에서는 내보내기 기능이 제한적입니다. 중요한 내용은 복사하여 별도 보관하세요.

ADK(선택)
- ADK를 사용하면 로컬 ADK 서버로 에이전트를 실행할 수 있습니다. 설정에서 ADK를 활성화하면 상단에 상태가 표시됩니다.

설정
- Settings에서 기본 모델, 엔드포인트, ADK 관련 설정(파이썬 경로) 및 API 키를 구성할 수 있습니다.

개인정보 및 보안
- 대화와 메시지는 사용자의 로컬 디스크에 저장됩니다.
- 외부 LLM 또는 ADK를 사용하도록 구성하지 않으면 데이터는 외부로 전송되지 않습니다.

자주 묻는 질문
- Q: 아이콘이나 앱 이름이 기본으로 보입니다.  
  A: 설치된 패키지에 포함된 아이콘/브랜딩을 사용합니다. 필요하면 배포자에게 문의하세요.

- Q: ADK를 켰는데 응답이 실패합니다.  
  A: 상단의 ADK 상태를 확인하고, 문제가 계속되면 ADK를 끄고 기본 LLM으로 시도하세요.

지원
- 설치 파일, 버그 리포트, 기능 요청은 GitHub Releases 또는 Issues를 통해 남겨주세요. 운영체제(OS)와 간단한 상황 설명을 포함하면 빠른 대응에 도움이 됩니다.
